Privacy

TL;DR: thescanner keeps every byte of your data on the device that runs it. No analytics, no telemetry, no third-party SDKs.

What's stored locally

  • Your scanner-server endpoints, tokens, and scan settings — under the app's private data directory.
  • Each scan you start is saved as a JSON file containing the IPs you supplied plus their per-resolver test results.
  • A short in-memory ring of recent log lines for the live-log pane. Not persisted to disk.

What's sent over the network

  • DNS queries to the resolvers you chose to scan. The queries are encrypted with the shared token of the scanner-server you chose to talk to.
  • Optional "cover-traffic" DNS lookups to well-known public hostnames (google.com, wikipedia.org, …) when noise is enabled. Off by setting scan.noise_enabled = false.

No telemetry

thescanner does not phone home. It contains no analytics SDKs, no crash-reporting SDKs, and no advertising IDs. Updates are not auto-fetched; the app checks GitHub for new releases only when you ask it to.

Permissions

The mobile builds use only the Internet permission. They don't request the camera, microphone, contacts, photos, or location.

Questions or removal requests

File an issue at the GitHub repo linked from About.
